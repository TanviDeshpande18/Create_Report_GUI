Sequencing quality - per base quality

The Illumina NextSeq 550 assigns a quality score to every read.

This is an estimation of the accuracy of base calling while performing the sequencing by synthesis reaction in the flow-cell. This value is measured by the instrument while reading every synthesis reaction.

The quality values are reported along with the data. The values are extracted from the data using a tool called fastqc.

Figure 1: Per base sequencing quality

This figure shows the per base sequencing quality of the reads as calculated using the as per the Illumina standard as defined here

The figure depicts average per base sequencing quality as a boxplot of values at each base in the read. This serves as a guide for pre-processing of the reads for quality filtration and read trimming.

A phred33 score over 30 implies a base call accuracy of 99.9%. For all the positions, 90% of the reads have a quality score above Q30 and thus pass the QC check for further analysis

Sequencing quality - Adapter contamination

Illumina uses adapter sequences - neutral DNA sequences included in the sequencing chemistry to enable annealing of the target strand to the flow cell adapters. In some cases, these sequences may remain in the final processed sample and could result in being over-represented.

This adapter sequence is of no use in the sample analysis and can decrease the potential usable information from any NGS experiment. To estimate this, fastqc checks each individual sequencing read for the presence of a list of standard adapter sequences used by Illumina.

The figure below represents amount of adapter sequences / 13 bases averaged over each sequencing read in the sample.

Sequencing accuracy estimation


The phiX174 DNA control is a sample with known sequence. Each read is thus aligned with the reference sequence using bowtie2 and the most stringent alignment parameters.

Bowtie2 parameters :

  • Reporting parameters : –all reports all possible alignments

Alignment parameters:

  • End to end alignment mode. Thus ensures that entire reads are considered while calculating alignments.

Mismatch penalty calculation function

–very-sensitive. Where:

  • D 20 -R 3 -N 0 -L 20 -i S,1,0.50
  • D - The minimum matching length of initial seed 25% of the read length.
  • R - The offset between two consecutive alignments is 3 bases.
  • N - Zero iterations allowed to extend the read alignment.
  • L - Minimum read length that needs to align for a match to be called.
  • i Mismatch calculation scales as a function of square of misaligned length

This alignment mode of bowtie2 allows for find exact matching alignments that can harbour upto 4 mismatches in a read of length 120

Per base sequencing accuracy


Sequencing coverage is defined as :

Number of total bases sequenced / Number of bases in the reference sequence [1]


Number of total bases sequenced can be calculated as :

Number of bases per sequencing reads x Total number of sequences reads [2]


Parameter Calculation Value
Number of total bases sequenced (Formula [2]) {read_length} x {bases_sequenced_bp} {total_number_of_bases} bases

That is {bases_sequenced_in_Mb} Megabases of DNA sequence information.

Thus the expected coverage based on Formula [1] is:

{total_number_of_bases}/ {genome_size_in_bp} = {expected_coverage} X coverage of the phiX174 reference genome.


Sequencing coverage per reference is calculated using alignment data from the output of bowtie2.

A sliding window of length = {read_length} bases is used to measure the number of reads in that window, with an offset of 40 bases. Reads that occur in adjacent windows are only counted in the first window they occur in.

A non uniform amplification of the DNA library could result into skewed sequencing coverage along the length of the reference genome. A non uniform coverage could lead to copy number estimation errors and assignment of confidence values in genome assembly.
